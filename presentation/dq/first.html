UI / UX

데이터셋을 엑셀, CSV 뿐만 아니라 DB에서도 임포트 가능

(Documentation을 보면 심지어 NoSQL도 가능하다.)

Join, Aggregate 등을 이용하여 query 질의문 같이 데이터 전처리 가능

(SELECT 같은 경우는 Select Attribute를 사용하면 된다.)

Generate Attribute 오퍼레이터를 통해 새로운 로직의 Attribute를 만들 수 있음

(기본적인 문법을 제공해줘 코딩을 모르더라도 어느정도 사용할 수 있을듯 보인다.)

각 오퍼레이터 노드들을 이해하는게 난해

(Documentation과 내용이 똑같다.)

각 오퍼레이터들의 인풋 아웃풋 이름이 통일되어 있지 않음

(같은 Example Set이라도 출력 방식이 서로 다르다.)

데이터를 다루는게 제한적임.

(DeepLearning 노드 같은경우 Label 칼럼이 필수적)

공식과 원리를 이해하고 있더라도 즉각적인 구현이 어려울 수 있음.

성능

큰 데이터를 임포트시키면 프로그램 전체가 무거워짐

(lazy하게 로딩하지 않고 그때그때 바로 가지고 옴)

용량이 큰 데이터셋을 수정하거나 열람시 시스템 전체가 느려짐

(프로그램약 뿐만 아니라 전반적인 OS전체가 느려진다.)

요약

  • 큰 데이터셋을 다룰때 사용성이 급격하게 떨어짐
  • 빈약한 튜토리얼, 질 좋은 레거시 모델의 부재로 학습비용이 높음
  • 코딩과의 작업플로우가 다름
  • 꽤 괜찮은 데이터 처리 방식
